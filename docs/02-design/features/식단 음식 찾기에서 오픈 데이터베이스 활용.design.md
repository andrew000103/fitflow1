## 1. 개요

Plan의 4가지 기능 요구사항을 **Option C (Pragmatic Balance)** 방식으로 구현한다.

- `diet-search.ts`에 `saveMealItemFromFoodItem` 함수 1개 추가
- `food-search-screen.tsx`의 `FoodRow` 의존성을 전체 `FoodItem`으로 전환
- 출처 뱃지 및 "더 보기" UI를 `FoodResultCard` 및 `FlatList`에 인라인 추가
- 새 파일 생성 없음, Supabase 스키마 변경 없음

### 3.2 saveMealItemFromFoodItem 함수

```typescript
const UUID_REGEX = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;

export async function saveMealItemFromFoodItem(params: {
  userId: string;
  foodItem: FoodItem;
  mealType: string;
  date: string;
  amountG: number;
}): Promise<void> {
  const { userId, foodItem, mealType, date, amountG } = params;
  const ratio = amountG / 100;
  const n = foodItem.nutrients;

  const { error } = await supabase.from('meal_items').insert({
    user_id: userId,
    food_id: UUID_REGEX.test(foodItem.id) ? foodItem.id : null,
    meal_type: mealType,
    date,
    amount_g: amountG,
    calories: Math.round(n.calories * ratio),
    protein_g: Math.round(n.protein_g * ratio * 10) / 10,
    carbs_g: Math.round(n.carbs_g * ratio * 10) / 10,
    fat_g: Math.round(n.fat_g * ratio * 10) / 10,
    sodium_mg: n.sodium_mg != null ? Math.round(n.sodium_mg * ratio * 10) / 10 : null,
    sugar_g: n.sugar_g != null ? Math.round(n.sugar_g * ratio * 10) / 10 : null,
  });

  if (error) throw error;
}
```

**설계 결정:**
- `UUID_REGEX`: UUID 형식이면 `food_id`에 저장, 아니면 `null`
  - custom 음식 (Supabase UUID) → food_id 저장됨
  - MFDS (`mfds:name:0`), OFFs (바코드), USDA (`usda:12345`) → food_id = null
- 기존 `isTempId('off::...')` 함수와는 별도로, 더 범용적인 UUID 체크 방식 사용
- 영양소 계산: `FoodItem.nutrients.*` 은 모두 per 100g 기준 (기존 `FoodRow.*_per_100g`과 동일)

### 4.3 doSearch 변경

```typescript
const doSearch = useCallback(async (q: string, pg = 1) => {
  const normalized = q.trim();
  if (!normalized) {
    setResults([]);
    setHasSearched(false);
    return;
  }
  if (pg === 1) {
    setLoading(true);
    setHasSearched(false);
  } else {
    setLoadingMore(true);
  }
  try {
    const items = await searchFoods(normalized, pg, user?.id ?? undefined);
    if (pg === 1) setResults(items);
    else setResults(prev => [...prev, ...items]);
    setPage(pg);
  } catch {
    if (pg === 1) setResults([]);
  } finally {
    if (pg === 1) {
      setLoading(false);
      setHasSearched(true);
    } else {
      setLoadingMore(false);
    }
  }
}, [user?.id]);
```

**쿼리 변경 시 page 초기화:**


## 5. 엣지 케이스 처리

| 케이스 | 처리 방법 |
|--------|---------|
| OFFs ID (바코드): `food_id` null | UUID_REGEX 불일치 → `food_id = null` 저장 |
| MFDS/USDA ID: `food_id` null | 동일, `food_id = null` |
| `nutrients.sodium_mg` 없는 경우 | `null` 그대로 전달 (기존 동작 동일) |
| "더 보기" 클릭 시 결과 없으면? | `items.length === 0` → append 없음, 버튼은 그대로 표시 |
| 검색어 변경 시 더 보기 중복 클릭 | `loadingMore` 체크로 방지 |
| 직접 입력 후 selectedFood(FoodItem) 설정 | `toFoodItem(FoodRow)` 변환 유지 |

# 기능 설계: 운동 시각 자료 안내

**기능명:** `exercise-visual-guide`
**버전:** 1.0.0
**날짜:** 2026-04-01

## 1. 개요

이 문서는 사용자가 올바른 운동 자세를 배울 수 있도록 GIF 또는 유튜브 영상을 제공하는 '운동 시각 자료 안내' 기능의 설계를 다룹니다. 이 기능은 사용자의 자신감을 높이고, 정확한 자세를 유도하며, 전반적인 사용자 경험을 향상시키는 것을 목표로 합니다.

## 2. 아키텍처 설계

### 2.1. 데이터 스키마

`exercises` 테이블에 시각 자료의 URL을 저장하기 위한 새로운 열을 추가합니다.

**테이블: `exercises`**

| 열 이름 | 데이터 타입 | 설명 |
|---|---|---|
| `id` | `uuid` | 기본 키 |
| `name` | `text` | 운동 이름 |
| ... | ... | ... |
| `visual_guide_url` | `text` | GIF 또는 유튜브 영상의 URL |

이 간단한 접근 방식을 통해 데이터베이스 구조에 큰 변경 없이 각 운동과 시각 자료를 쉽게 연결할 수 있습니다.

### 2.2. 콘텐츠 수집 및 관리

-   **콘텐츠:** 주로 신뢰할 수 있는 출처의 고품질, 짧고 반복되는 GIF를 수집합니다. GIF가 부적절한 운동의 경우, 특정 유튜브 영상으로 연결합니다.
-   **선별:** 각 운동에 가장 적합한 시각 자료를 수동으로 선별하여 정확하고, 명확하며, 안전한지 확인합니다.
-   **저장소:** GIF는 빠른 검색을 위해 전용 클라우드 저장소 버킷(예: S3 또는 Google Cloud Storage)에 저장합니다. 유튜브 링크는 텍스트 URL로 저장됩니다.
-   **관리 도구:** 각 운동의 `visual_guide_url`을 쉽게 업데이트할 수 있는 간단한 내부 도구나 스크립트를 개발합니다.

### 2.3. 프론트엔드 아키텍처

-   **컴포넌트:** 새로운 리액트 네이티브 컴포넌트인 `ExerciseVisualGuide`를 생성합니다.
    -   `visual_guide_url`을 prop으로 받습니다.
    -   URL이 GIF인 경우, 리액트 네이티브의 `<Image>` 컴포넌트를 사용해 이미지를 표시합니다.
    -   URL이 유튜브 영상인 경우, 썸네일을 표시하고 탭하면 `Linking.openURL()` 또는 `react-native-webview`와 같은 인앱 브라우저를 사용하여 영상을 엽니다.
-   **상태 관리:** `visual_guide_url`은 다른 운동 정보와 함께 가져와 기존의 운동/운동 상태 저장소(예: Zustand 또는 Redux) 내에서 관리됩니다.
-   **UI 통합:** `ExerciseVisualGuide` 컴포넌트는 운동 상세 화면에 통합되며, 운동 이름 옆이나 지정된 미디어 영역에 표시될 가능성이 높습니다.

## 3. UI/UX 디자인

### 3.1. 목업

**운동 상세 화면 - 변경 전**
(텍스트만 있는 간단한 레이아웃)
-   운동 이름
-   세트/반복/무게 정보

**운동 상세 화면 - 변경 후**
-   [썸네일 이미지] 운동 이름
-   세트/반복/무게 정보

*(썸네일 이미지는 GIF 또는 유튜브 영상의 작고 눈에 띄지 않는 미리보기가 될 것입니다.)*

**탭/클릭 시:**
-   **GIF:** GIF를 더 큰 전체 너비 형식으로 표시하는 모달이 열립니다. 닫기 버튼이 있습니다.
-   **유튜브 영상:** 특정 영상으로 유튜브 앱이 실행되거나, 영상 URL로 인앱 웹뷰가 열립니다. 둘 중 어느 것이 더 나은 사용자 경험을 제공하는지에 따라 구현 시 결정됩니다.

### 3.2. 사용자 흐름

1.  사용자가 운동 상세 정보 화면으로 이동합니다.
2.  앱은 운동 이름, 세부 정보 및 시각 자료 썸네일을 표시합니다.
3.  사용자가 썸네일을 탭합니다.
4.  GIF인 경우, 애니메이션 GIF가 포함된 모달이 나타납니다. 사용자는 이를 닫을 수 있습니다.
5.  유튜브 링크인 경우, 사용자는 시연을 보기 위해 해당 영상으로 이동합니다. 사용자는 앱으로 다시 돌아올 수 있습니다.

## 4. 구현 세부사항

-   **백엔드:** 초기 운동 세트에 대한 `visual_guide_url`을 채우기 위해 일회성 스크립트를 실행합니다.
-   **프론트엔드:**
    -   `ExerciseVisualGuide` 컴포넌트를 생성합니다.
    -   이 새로운 컴포넌트를 포함하도록 운동 화면을 수정합니다.
    -   `visual_guide_url`을 포함하도록 데이터 가져오기 로직을 업데이트합니다.
    -   GIF용 모달과 유튜브 영상용 링크를 구현합니다.
-   **라이브러리:** 더 나은 GIF 성능과 캐싱을 위해 `react-native-fast-image`를 사용할 수 있으며, 유튜브 영상용 인앱 브라우저를 결정할 경우 `react-native-webview`를 사용할 수 있습니다.

## 5. 성공 지표 (계획에서 가져옴)

-   사용자 참여도 (시각 자료 조회 수)
-   정성적 피드백 (설문조사, 리뷰)
-   사용자 유지율
-   기능 채택률

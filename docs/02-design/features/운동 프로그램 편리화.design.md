## 1. 개요

Plan의 3가지 기능 요구사항을 **Option C (인라인 로컬 컴포넌트)** 방식으로 구현한다.

- `WorkoutScreen` 파일 내에 `WeeklyStrip`, `TodayPreviewPanel` 로컬 컴포넌트를 정의
- `ProgramCreateScreen` 파일 내에 `moveExercise` 함수만 추가
- 새 파일 생성 없음, Supabase 스키마 변경 없음

## 3. workout-screen.tsx 설계

### 3.1 추가 State

```typescript
// 미리보기 패널 관련
const [previewExercises, setPreviewExercises] = useState<ProgramExerciseRow[]>([]);
const [previewDay, setPreviewDay] = useState<number>(1);   // 선택된 Day (기본: current_day)
const [previewExpanded, setPreviewExpanded] = useState(true);
const [previewLoading, setPreviewLoading] = useState(false);
const [userTMs, setUserTMs] = useState<Record<string, number>>({});
```

### 3.2 데이터 로딩 흐름

```
useFocusEffect (기존 fetchActiveProgram 유지)
  └── fetchActiveProgram() → activeUserProgram 로드
       └── (activeUserProgram 존재 시) loadPreview(current_day) 호출

loadPreview(programId: string, userProgramId: string, dayNum: number, programName: string)
  ├── setPreviewLoading(true)
  ├── fetchProgramDayExercises(programId, dayNum)
  │    └── setPreviewExercises(result)
  ├── (isNsunsProgram(null, programName)) fetchUserTMs(userProgramId)
  │    └── setUserTMs(result)
  └── setPreviewLoading(false)
```

> `loadPreview`는 `useCallback`으로 메모이제이션. 4개 파라미터를 명시적으로 전달받아 클로저 의존성 없이 호출 가능.
> `useFocusEffect` 내에서는 `useProgramStore.getState().activeUserProgram`으로 최신 상태를 직접 읽어 호출.

### 3.3 로컬 컴포넌트: WeeklyStrip

**위치**: `WorkoutScreen` 바깥, 파일 상단에 정의 (파일 내 로컬)

```typescript
function WeeklyStrip({
  daysPerWeek,
  currentDay,
  selectedDay,
  onDayPress,
}: {
  daysPerWeek: number;
  currentDay: number;
  selectedDay: number;
  onDayPress: (day: number) => void;
}) { ... }
```

**렌더링 로직**:
- `Array.from({ length: daysPerWeek }, (_, i) => i + 1)` 으로 Day 목록 생성
- 각 Day 카드 상태:
  - `day < currentDay` → 완료 (✓, 흐린 accent 배경)
  - `day === currentDay` → 진행중 (▶, 강조 accent 배경 + border)
  - `day > currentDay` → 예정 (회색 배경)
  - `day === selectedDay` → 선택됨 (border 강조, selectedDay ≠ currentDay인 경우)
- `ScrollView horizontal showsHorizontalScrollIndicator={false}` 사용
- 각 카드: `TouchableOpacity`, 너비 56, 높이 64, borderRadius 12

**카드 내부 레이아웃**:
```
┌─────────┐
│  D{n}   │  ← 볼드 텍스트
│  ✓/▶/○  │  ← 상태 아이콘
└─────────┘
```

### 3.4 로컬 컴포넌트: TodayPreviewPanel

**위치**: `WeeklyStrip` 아래 정의

```typescript
function TodayPreviewPanel({
  exercises,
  isNsuns,
  tms,
  selectedDay,
  currentDay,
  loading,
  expanded,
  onToggle,
}: {
  exercises: ProgramExerciseRow[];
  isNsuns: boolean;
  tms: Record<string, number>;
  selectedDay: number;
  currentDay: number;
  loading: boolean;
  expanded: boolean;
  onToggle: () => void;
}) { ... }
```

**헤더 행**:
- 왼쪽: "Day {selectedDay} 종목" 텍스트
- 오른쪽: chevron-up / chevron-down 아이콘 (expanded 상태)
- `TouchableOpacity`로 onToggle 연결

**종목 행 (expanded 시)**:
```
종목명              N세트 × M회 @ Wkg
(nSuns) 종목명     9세트 (TM: Wkg)
```

nSuns 중량 표시:
```typescript
function getNsunsWeightLabel(ex: ProgramExerciseRow, tms: Record<string, number>): string {
  const tmKey = getTmKey(ex.exercises?.name_ko ?? '', ex.exercises?.name_en ?? null);
  if (!tmKey || !tms[tmKey]) return `${ex.target_sets}세트 × ${ex.target_reps}회`;
  return `${ex.target_sets}세트 (TM: ${tms[tmKey]}kg)`;
}
```

비어있을 때: "종목 정보 없음" (textTertiary 색상)
로딩 중: `ActivityIndicator` 표시

### 3.5 WorkoutScreen 렌더 트리 변경

기존 `activeUserProgram` 카드 블록 내부에 두 컴포넌트 삽입:

```
<View programCard>
  <View programCardTop>          ← 기존 (프로그램명, ▶ 버튼)
  <View progressTrack>           ← 기존 (진행 막대)

  ─── NEW ───────────────────
  <WeeklyStrip                   ← 추가 (주간 스케줄)
    daysPerWeek={...}
    currentDay={...}
    selectedDay={previewDay}
    onDayPress={handleDaySelect}
  />
  <TodayPreviewPanel             ← 추가 (오늘 종목)
    exercises={previewExercises}
    isNsuns={isNsuns}
    tms={userTMs}
    selectedDay={previewDay}
    currentDay={activeUserProgram.current_day}
    loading={previewLoading}
    expanded={previewExpanded}
    onToggle={() => setPreviewExpanded(v => !v)}
  />
  ─── END NEW ────────────────

  <TouchableOpacity todayBtn>    ← 기존 (오늘의 운동 시작 버튼)
</View>
```

**handleDaySelect**:
```typescript
const handleDaySelect = useCallback(async (day: number) => {
  setPreviewDay(day);
  await loadPreview(day);
}, [loadPreview]);
```

## 4. program-create-screen.tsx 설계

### 4.1 moveExercise 함수

기존 `days` state (`DraftDay[]`)를 직접 조작:

```typescript
const moveExercise = useCallback((dayIdx: number, exIdx: number, dir: 'up' | 'down') => {
  setDays(prev => {
    const updated = prev.map(d => ({ ...d, exercises: [...d.exercises] }));
    const exs = updated[dayIdx].exercises;
    const targetIdx = dir === 'up' ? exIdx - 1 : exIdx + 1;
    if (targetIdx < 0 || targetIdx >= exs.length) return prev;
    [exs[exIdx], exs[targetIdx]] = [exs[targetIdx], exs[exIdx]];
    return updated;
  });
}, []);
```

### 4.2 UI 변경: 종목 행

기존 종목 행 우측에 ▲/▼ 버튼 추가:

```
┌─────────────────────────────────────────┐
│ [종목명]           [▲] [▼] [🗑️]        │
│ 세트 × 반복 @ 중량                      │
└─────────────────────────────────────────┘
```

구현:
- `▲` 버튼: `exIdx === 0` 이면 `opacity: 0.3, disabled`
- `▼` 버튼: `exIdx === day.exercises.length - 1` 이면 `opacity: 0.3, disabled`
- 아이콘: `MaterialCommunityIcons name="chevron-up"` / `"chevron-down"`, size 20
- 버튼 `hitSlop={{ top: 8, bottom: 8, left: 8, right: 8 }}`

## 5. 엣지 케이스 처리

| 케이스 | 처리 방법 |
|--------|----------|
| 활성 프로그램 없음 | `activeUserProgram === null` → WeeklyStrip/TodayPreviewPanel 렌더링 안 함 (기존 동작 유지) |
| nSuns + TM 미설정 | `tms[tmKey]` 없을 시 "TM 설정 필요" 텍스트 표시 (빨간색) |
| 해당 Day에 종목 없음 | "종목 정보 없음" 텍스트 표시 |
| previewDay가 current_day와 다를 때 "오늘의 운동 시작" 버튼 | 버튼 레이블을 "Day {currentDay} 시작"으로 고정 (previewDay 선택과 무관) |
| Expo Go 최초 접속 시 데이터 없음 | previewLoading spinner 표시 후 결과 렌더링 |

# [PLAN] AI 플랜 운동 직접 선택 기능

- **Author:** Gemini
- **Date:** 2026-04-01
- **Status:** DRAFT

---

## 1. Executive Summary

본 문서는 AI가 생성한 운동 계획에서 사용자가 운동 종목을 변경할 때, 기존의 추천 종목 리스트 외에 직접 원하는 운동을 검색하고 선택할 수 있는 기능을 추가하는 개발 계획을 정의합니다. 현재 시스템은 몇 가지 대안 운동만 제시하여 사용자의 선택권이 제한적입니다. 이 기능은 사용자에게 완전한 통제권과 유연성을 제공하여 개인의 선호도, 운동 환경, 또는 특정 제약 조건에 맞춰 계획을 수정할 수 있게 합니다. 이를 통해 사용자 만족도를 극대화하고 서비스의 개인화 수준을 한 단계 높이고자 합니다.

- **Problem:** 운동 종목 변경 시, 시스템이 제안하는 제한된 추천 목록에서만 선택할 수 있어 사용자의 자유로운 선택이 불가능합니다.
- **Solution:** 추천 운동 목록 하단에 '직접 운동 선택' 옵션을 추가하고, 사용자가 전체 운동 데이터베이스에서 원하는 운동을 검색하여 교체할 수 있는 인터페이스를 제공합니다.
- **Core Value:** 사용자 선택권 및 통제권 강화, 계획의 유연성 증대, 개인화 경험 향상.

## 2. Problem Statement

- **What:** AI 운동 계획에서 특정 운동을 다른 것으로 교체하고 싶을 때, 시스템이 제공하는 몇 가지 추천 운동으로만 변경이 가능합니다.
- **Where:** 운동 교체 화면 (`SwapExerciseSheet.tsx`).
- **When:** 사용자가 추천 목록에 없는 특정 운동(예: 다른 기구 사용, 맨몸 운동)으로 변경하고 싶을 때.
- **Impact:** 사용자는 원하는 운동으로 계획을 수정할 수 없어 만족도가 저하되거나, 수동으로 계획을 기억하고 실행하는 불편함을 겪습니다. 이는 장기적으로 서비스 이탈로 이어질 수 있습니다.

## 3. Goal Statement

- **Primary Goal:** 사용자가 AI 플랜의 특정 운동을 교체할 때, 앱에 등록된 전체 운동 목록에서 원하는 운동을 직접 검색하고 선택하여 변경할 수 있도록 시스템을 개선합니다.
- **Secondary Goal:** 운동 검색 및 선택 인터페이스를 다른 기능에서도 재사용할 수 있도록 모듈식으로 설계합니다.

## 4. Success Metrics

- **Quantitative:**
  - 운동 교체 기능 사용 시, '직접 선택' 옵션의 사용률이 30% 이상 기록.
  - 기능 출시 후, 운동 계획 수정 기능 관련 부정적 피드백 50% 감소.
- **Qualitative:**
  - "원하는 운동으로 바꿀 수 있어서 훨씬 편하고 좋아요"와 같은 긍정적인 사용자 피드백.
  - 새롭게 추가된 운동 검색/선택 플로우가 직관적이고 사용하기 쉽다는 평가.

## 5. Scope

### In-Scope
- 운동 교체 화면(`SwapExerciseSheet.tsx`)에 '직접 운동 검색/선택' UI 요소 추가.
- 전체 운동 목록을 검색하고 선택할 수 있는 별도의 검색 화면 또는 모달 개발.
- 사용자가 선택한 운동으로 기존 운동 계획의 상태를 업데이트하는 로직 구현 (`ai-plan-store.ts` 또는 `workout-store.ts`).
- 교체된 운동 정보가 운동 세션 화면에 올바르게 반영되는지 확인.

### Out-of-Scope
- 새로운 운동 종목을 데이터베이스에 추가하는 기능.
- 운동 추천 알고리즘 자체의 변경.
- 운동 교체와 관련 없는 다른 화면의 대대적인 UI/UX 개편.

**Total Estimated Time:** 5-6 days

## 7. Dependencies & Risks

- **Dependencies:**
  - 검색 가능한 전체 운동 목록을 제공하는 API 또는 데이터 소스가 필요함 (`src/lib/exercise-utils.ts` 또는 유사 파일).
- **Risks:**
  - 운동 검색 UI가 복잡해지면 사용성이 저하될 수 있음. (완화: 자동 완성, 필터링 등 사용자 친화적인 검색 기능 제공)
  - 상태 관리 로직이 복잡하여 예기치 않은 사이드 이펙트를 유발할 수 있음. (완화: `zustand`와 같은 상태 관리 라이브러리의 원칙에 따라 명확하게 로직을 분리하고, 철저한 테스트 수행)

# AI 플랜 UX 개선 및 신뢰도 향상 계획

## 1. 목표 (Objective)

AI가 생성하는 운동 계획의 사용자 경험을 개선하고, 사용자가 AI 플랜을 더 신뢰할 수 있도록 신뢰도를 높이는 것을 목표로 합니다.

## 2. 범위 (Scope)

- **주 운동 일수 세분화:**
    - 사용자가 AI 플랜 생성 시 주 2, 3, 4, 5, 6일 중 원하는 운동 횟수를 선택할 수 있는 UI를 제공합니다.
    - 선택된 운동 횟수에 맞춰 AI 플래너가 соответствующее 운동 계획을 생성하도록 로직을 수정합니다.

- **플랜 생성 이유 UI 개선:**
    - AI 플랜이 생성된 후, '플랜 생성 이유' 또는 'AI의 추천 근거' 섹션을 기본적으로 접힌 상태(collapsed)로 제공합니다.
    - 사용자가 궁금해할 경우, 해당 섹션을 펼쳐(expanded) 상세 내용을 확인할 수 있도록 토글 기능을 구현합니다.

- **AI 플랜 신뢰도 향상 방안 연구:**
    - 사용자들이 AI가 제안하는 운동 계획을 더 신뢰하고 따를 수 있도록 만드는 추가적인 장치를 구상하고 제안합니다.
    - 예시 아이디어:
        - **전문가 인용/근거 제시:** "이 계획은 NSCA의 근력 훈련 가이드라인을 기반으로 합니다" 와 같이 공신력 있는 출처나 원칙을 명시합니다.
        - **예상 효과 시각화:** "이 프로그램을 12주간 수행하면 약 2kg의 근육량 증가와 5kg의 벤치프레스 증량을 기대할 수 있습니다" 와 같이 구체적인 예상치를 제공합니다. (데이터 기반 추정)
        - **대체 운동 제안의 용이성:** 특정 운동이 마음에 들지 않거나 수행하기 어려운 경우, 쉽게 다른 추천 운동으로 교체할 수 있는 기능을 강화하고, 이를 통해 사용자에게 통제권을 부여합니다.
        - **사용자 피드백 반영:** "OO님의 피드백을 반영하여 플랜을 수정했습니다" 와 같이 사용자의 입력을 AI가 고려하고 있음을 보여주는 인터랙션을 추가합니다.

## 3. 제약 조건 (Constraints)

- 기존 애플리케이션 아키텍처 내에서 변경이 이루어져야 합니다.
- AI 계획 모델이 다양한 운동 빈도를 지원할 수 있어야 합니다.

## 4. 가정 (Assumptions)

- 사용자는 자신의 스케줄에 더 많은 유연성을 원합니다.
- 계획의 근거를 제공하면 사용자의 신뢰도와 준수율이 높아질 것입니다.
- 더 깨끗하고 압도적이지 않은 초기 프레젠테이션이 바람직합니다.

## 5. 성공 기준 (Success Criteria)

- 사용자가 주 2, 3, 4, 5, 6일 운동 계획을 성공적으로 생성할 수 있습니다.
- 계획 근거는 기본적으로 숨겨져 있으며 사용자가 확장할 수 있습니다.
- 신뢰도 향상을 위한 구체적이고 실행 가능한 아이디어 목록이 문서화됩니다.

## 6. 다음 단계 (Next Step)

- UI 및 로직 변경에 대한 상세 목업 및 기술 사양을 작성하기 위해 디자인 단계 (`/pdca design`)로 진행합니다.

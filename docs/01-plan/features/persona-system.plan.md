# [Plan] 페르소나 캐릭터 시스템 (Persona Character System)

## 1. 개요 (Executive Summary)
사용자의 운동 기록과 식단 데이터를 분석하여 고유한 '페르소나 캐릭터'와 '별명'을 부여함으로써 앱 사용에 재미(Amusement)와 정서적 유대감을 더합니다. 단순한 수치 제공을 넘어 사용자의 현재 상태를 시각적으로 피드백하여 리텐션을 강화합니다.

| 분류 | 내용 |
|---|---|
| **문제 정의** | 단순 수치(칼로리, 무게) 위주의 피드백은 딱딱하고 장기적인 동기부여가 부족함 |
| **해결 방안** | 사용자의 행동 패턴(운동 빈도, 식단 성격, 목표)을 분석하여 캐릭터와 별명을 부여 |
| **기능 UX 효과** | 홈 화면에서 나를 닮은 캐릭터가 현재 상태를 표현하여 앱과의 유대감 형성 |
| **핵심 가치** | 재미(Amusement), 동기부여(Motivation), 개인화(Personalization) |

## 2. 페르소나 유형 및 로직 (Persona Typology)

### A. 핵심 차원 (Core Dimensions)
1. **목표/성향 (Goal):** 증량(Gain) / 유지(Maintain) / 감량(Loss)
2. **성실도 (Frequency):** 하드코어(Hardcore, 주 5회+) / 꾸준함(Steady, 주 2~4회) / 간헐적(Sporadic, 주 1회 이하)
3. **식단 성격 (Diet):** 단백질 중시(Protein) / 균형 잡힌(Balanced) / 치팅 빈번(Cheater)

### B. 페르소나 예시 (Preset Types)
- **정진하는 고릴라:** 증량 목표 + 하드코어 운동 + 고단백 식단
- **우아한 사슴:** 감량 목표 + 꾸준한 운동 + 균형 잡힌 식단
- **자유로운 영혼의 나무늘보:** 유지 목표 + 간헐적 운동 + 치팅 빈번
- *(사용자가 그려올 일러스트에 맞춰 총 9~12종의 프리셋 정의)*

### C. 당일 상태 피드백 (Daily States)
동일한 페르소나 내에서도 오늘의 기록에 따라 일러스트 상태 변화:
- **Active:** 오늘 운동 완료 + 식단 목표 달성 (활기찬 모습)
- **Resting:** 운동 쉬는 날 (편안한 모습)
- **Hungry:** 칼로리 섭취가 너무 낮을 때 (배고픈 모습)
- **Overwhelmed:** 목표 칼로리 크게 초과 (배부른 모습)

## 3. 기술적 구현 계획 (Technical Implementation)

### A. 데이터 분석 로직 (`src/lib/persona-utils.ts`)
- 최근 14일간의 `workout_sessions`와 `diet_entries`를 분석하여 페르소나 ID 산출.
- 주기적인 업데이트 (예: 앱 실행 시 또는 매일 첫 진입 시).

### B. 상태 관리 (`src/stores/persona-store.ts`)
- 현재 페르소나 유형 및 일러스트 상태 저장.
- 사용자 정의 별명(선택 사항) 기능 지원.

### C. UI/UX 통합 (`src/components/home/PersonaCharacter.tsx`)
- 홈 화면의 `CalorieRing` 중앙 또는 상단에 캐릭터 배치.
- 캐릭터 탭 시 현재 상태에 대한 '한마디' 말풍선 출력.
- `assets/images/persona/` 폴더에 일러스트 관리.

## 4. 단계별 진행 계획 (Milestones)

1. **[Design]** 페르소나 산출 점수표(Matrix) 및 상세 유형 정의 (12종)
2. **[Do]** `persona-utils.ts` 분석 로직 구현 및 Mock 데이터 테스트
3. **[Do]** `PersonaCharacter` 컴포넌트 개발 및 홈 화면 레이아웃 조정
4. **[Check]** 실제 사용자 데이터를 기반으로 적절한 페르소나가 부여되는지 검증
5. **[Act]** 사용자 피드백을 반영하여 캐릭터 멘트 및 상태 로직 고도화

## 5. 검증 계획 (Validation Plan)
- **데이터 정확성:** 의도한 운동/식단 패턴이 정확한 페르소나 ID로 변환되는지 단위 테스트.
- **UI 일관성:** 다양한 기기 크기에서 `CalorieRing`과 캐릭터의 조화 확인.
- **사용자 반응:** 별명과 캐릭터가 실제 사용자의 성향과 매칭된다고 느끼는지 확인.

## 1. 배경 및 문제 정의

### 1.1 현재 상태
- `WorkoutScreen`에 활성 프로그램 카드가 있지만, 오늘 무엇을 할지는 "오늘의 운동 시작" 버튼을 누르기 전까지 알 수 없다.
- 진행 막대가 `current_day / days_per_week` 기준이라 "이번 주 몇 번째인지"는 표시되나 주간 패턴(어떤 날 쉬었는지, 다음 날은 무엇인지)은 보이지 않는다.
- `ProgramCreateScreen`에서 날별 종목 목록이 `FlatList`로 구성되나, 순서 변경 UI가 없어 추가 순서가 곧 최종 순서가 된다.

### 1.2 목표
1. 운동 탭 메인에서 오늘 종목 목록을 expand/collapse로 미리 확인
2. D1~Dn 가로 스크롤 카드로 주간 스케줄 시각화
3. 프로그램 생성 화면에서 종목 위아래 이동 버튼 제공

## 2. 요구사항

### 2.1 기능 요구사항

#### FR-01: 오늘 운동 미리보기 패널 (최우선)
- 활성 프로그램 카드 하단에 "오늘의 종목" 섹션 추가
- 현재 `current_day`에 해당하는 `program_exercises`를 로드하여 표시
- 종목명, 세트 수, 반복 수, 중량(있을 경우)을 한 줄에 표시
- 기본 상태: 펼침(expanded). 사용자가 닫으면 접힘 상태 유지 (로컬 상태로 관리, 세션 재진입 시 초기화 허용)
- 데이터 없을 때: "종목 정보 없음" 텍스트 표시
- nSuns의 경우: TM 기반 중량 계산값을 표시하되, TM이 없으면 "TM 설정 필요" 표시

#### FR-02: 주간 스케줄 가로 카드
- 활성 프로그램 카드 내 미리보기 패널 상단에 배치
- D1~Dn(days_per_week)을 가로 스크롤로 표시
- 각 Day 카드 상태:
  - 완료(✓): `completed_sessions`로 추산. current_day 이전 = 완료
  - 진행중(▶): current_day = 현재
  - 예정( ): current_day 이후 = 예정
- Day 이름이 있으면(program_days.name) 카드에 표시, 없으면 "Day N"
- 탭 시 해당 Day 종목을 미리보기 패널에 표시 (선택적으로 다른 Day 예습 가능)

#### FR-03: 프로그램 생성 종목 순서 변경
- `ProgramCreateScreen`의 각 날(DraftDay) 내 종목 목록에 위아래 이동 버튼(▲/▼) 추가
- 첫 번째 종목의 ▲ 버튼, 마지막 종목의 ▼ 버튼은 비활성화(disabled)
- 순서 변경은 draft 상태(로컬)에서만 이루어지며, 저장 시 `order_index` 기준으로 Supabase에 저장
- drag-to-reorder는 이번 범위 외 (추후 개선으로 남김)

### 2.2 비기능 요구사항
- 미리보기 패널 데이터 로딩: `useFocusEffect` 내에서 기존 `fetchProgramDayExercises` 재활용
- 성능: 데이터를 이미 로드한 경우 중복 fetch 없이 캐시 활용
- 기존 화면 레이아웃 파괴 없음: 활성 프로그램 없는 사용자 UX 그대로 유지

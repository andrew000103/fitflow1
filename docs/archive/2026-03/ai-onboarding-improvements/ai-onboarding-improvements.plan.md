## 1. 배경 및 문제 정의

### 1.1 현재 상태

`ai-onboarding-screen.tsx`의 강도 프로필 입력 화면(`isStrengthStep`)은 스쿼트·벤치·데드리프트·OHP·바벨로우의 **현재 사용 중량**을 `kg` 단위로 직접 입력받는다. 그러나:

- **1RM(1회 최대 중량)** 개념을 모르는 입문~중급 사용자는 어떤 숫자를 입력해야 하는지 혼란스럽다.
- "모르는 항목은 비워두세요"라는 안내가 있으나, 실제로는 빈 값이 `strengthSkipped=true`와 동일한 효과(맨몸 기준)를 낸다.

`ai-plan-result-screen.tsx`에서 플랜을 다시 만들고 싶은 사용자는 현재:
- `AIOnboarding`으로 바로 이동해 전체 설문을 다시 해야 한다.
- 기존 답변을 유지한 채 플랜만 재생성하는 경로가 없다.

`plateauHistory` 질문("이전에 비슷한 목표에서 정체기를 겪은 적 있나요?")은:
- "정체기"라는 단어가 운동 맥락에서 익숙하지 않은 사용자에게 낯설다.
- 선지 값("식단 유지 실패", "운동 루틴 정체", "동기 부족")이 문어체·결과 진단 어조여서 공감도가 낮다.

### 1.2 목표

1. **1RM 계산기**: 강도 입력 화면에서 종목별 '1RM 계산' 버튼 → 미니 모달(무게 kg + 반복 횟수) → Epley 공식으로 추정 1RM을 해당 입력 필드에 자동 채움
2. **플랜 재생성 선택**: AIPlanResult 화면에 '플랜 다시 만들기' 버튼 → 바텀시트 "기존 정보로 재생성 / 새로 설문하기" 선택
3. **정체기 질문 개선**: 질문·선지 자연어 개선

## 2. 요구사항

### FR-1: 1RM 계산기 버튼 (강도 입력 화면)

- **위치**: `isStrengthStep` 화면의 각 종목 행 우측 (중량 입력 필드 옆)
- **UX 흐름**:
  1. '1RM 계산' 버튼 탭
  2. 해당 종목 이름이 타이틀인 모달 표시
  3. 입력 필드: 들었던 무게(kg) + 반복 횟수(회)
  4. **실시간 계산**: Epley 공식 `1RM = weight × (1 + reps / 30)` 결과를 하단에 표시
  5. '이 값으로 입력' 버튼 → 모달 닫힘 + 해당 `strengthInputs[ex.id]` 에 `Math.round(1RM)` 값 설정
- **공식**: Epley (국제적으로 가장 널리 쓰임) — `1RM = w × (1 + r/30)`
- **예외**: reps ≤ 0 또는 weight ≤ 0이면 계산 버튼 비활성화

### FR-2: 플랜 재생성 선택 (AIPlanResult)

- **위치**: `ai-plan-result-screen.tsx` 내 '플랜 다시 만들기' 버튼 (기존 버튼 또는 신규 추가)
- **UX 흐름**:
  1. '플랜 다시 만들기' 버튼 탭
  2. 바텀시트(Modal) 표시
  3. 선택지:
     - **기존 정보로 재생성**: 저장된 `onboardingData`로 바로 `generateAIPlan` 호출 → 로딩 화면 후 결과 갱신
     - **새로 설문하기**: `AIOnboarding` 화면으로 이동 (기존 동작)
  4. 취소 버튼 (바텀시트 닫기)
- **데이터**: `useAIPlanStore`의 `onboardingData` (이미 저장돼 있음) 재활용
- **조건**: `onboardingData`가 null이면 '기존 정보로 재생성' 옵션 비활성화 (또는 미표시)

### FR-3: 정체기 질문 개선

- **기존**:
  - 질문: "이전에 비슷한 목표에서 정체기를 겪은 적 있나요?"
  - 선지: 없어요 / 식단이 문제였어요 / 운동이 지루해졌어요 / 의지력이 떨어졌어요
- **변경**:
  - 질문: "운동이나 식단을 꾸준히 하다가 막힌 적이 있나요?"
  - 선지:
    - 없어요 (`없음`)
    - 식단 관리가 힘들었어요 (`식단 유지 실패`)
    - 루틴이 지겨워졌어요 (`운동 루틴 정체`)
    - 의욕이 떨어졌어요 (`동기 부족`)
  - value 값은 Supabase/AI 프롬프트 호환을 위해 **기존과 동일하게 유지**

## 3. 비기능 요구사항

- 1RM 계산기 모달은 기존 스타일(`colors`, `StyleSheet`)을 그대로 활용
- 바텀시트는 React Native `Modal`(기존 패턴) 사용, 외부 라이브러리 추가 없음
- Epley 공식 계산은 순수 JS (별도 라이브러리 불필요)
